Why this matters

CHERI changes the failure mode

Verifier researchers

Hardware bounds provide invariants that do not depend on perfect range analysis.

Kernel and runtime engineers

JIT mistakes can become traps or rejects instead of silent memory corruption.

Users and operators

Sandbox escape classes become harder to exploit even when software reasoning is incomplete.

This does not remove the verifier. It shows a route toward a verifier plus hardware enforcement model.
