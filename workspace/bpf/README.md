# Generated eBPF Fixtures

These C files are intentionally small inputs for `clang -target bpf`. They are
not application code. Each file compiles to an ELF eBPF object that the uBPF ELF
loader passes to the CHERI-aware JIT test harness.

The fixtures are split into two groups.

Positive cases should execute normally under CHERI:

- `stack_array.c`: dynamic bounded stack byte access
- `branch_stack.c`: branch-selected stack byte access
- `stack_widths.c`: bounded stack accesses at multiple scalar widths
- `context_in_bounds.c`: direct in-bounds context reads
- `branch_context_in_bounds.c`: branch-selected in-bounds context read
- `arithmetic_context_in_bounds.c`: arithmetic-derived in-bounds context read

Negative cases should trap under CHERI bounds:

- `context_oob.c`: direct out-of-bounds context read
- `branch_context_oob.c`: branch-selected out-of-bounds context read
- `arithmetic_context_oob.c`: arithmetic-derived out-of-bounds context read
- `stack_oob.c`: dynamic out-of-bounds stack access

Rebuild and inspect the generated bytecode with:

```sh
make -B compile-generated-bpf
```

Run the Morello/CheriBSD JIT suite with:

```sh
sg docker -c 'make run-cheri-generated-bpf'
```
